# AI 網球賽事智慧分析平台

---

## 作品摘要

本作品「AI 網球賽事智慧分析平台」是一套整合電腦視覺與大型語言模型的網球比賽影片分析系統。使用者只需上傳一段網球比賽影片，系統即可自動完成球體追蹤、球員姿態偵測、事件識別（擊球、反彈、回合切換）及統計彙整，並輸出帶有標注框架的分析影片與結構化 JSON 資料。分析結果涵蓋回合摘要、選手站位深度分布、球速統計（平均／最高／最低）及球場落點熱力圖等五大維度，協助教練與選手快速掌握比賽關鍵數據。

此外，平台搭載多模態 AI 聊天功能，以 vLLM 框架在本地運行 Qwen 多模態語言模型，使用者可直接針對影片內容提問，系統將以解說員視角即時串流回應，實現「看完即問、即問即答」的互動體驗。

系統採前後端分離架構：前端以 Next.js 搭配 React 19 建構現代化單頁應用，後端以 FastAPI 提供高效能非同步 API，結合 MySQL 持久化儲存，並透過 Docker Compose 實現一鍵部署，搭配 Cloudflare Tunnel 對外提供 HTTPS 安全服務。平台同時支援訪客模式與會員模式，訪客無需註冊即可體驗完整分析功能，影片資料七天後自動清除，兼顧易用性與資源管理。

本系統將傳統需要大量人工觀看的賽事回顧工作，轉化為秒級自動化處理流程，大幅降低教練、選手與賽事分析人員的作業成本，具備明確的產業落地價值。

## 作品說明書

### 一、創意發想背景及概述

網球運動的訓練與賽事分析，長期依賴教練以肉眼觀察、人工記錄的方式進行，不僅耗時，且容易受限於主觀判斷。隨著 GPU 運算成本下降與 YOLO 系列物件偵測模型的成熟，電腦視覺技術已逐漸應用於運動科學領域，但現有工具多為封閉商業產品，缺乏整合 AI 對話的互動性，且部署門檻高，不易由非技術人員使用。

本專案發想源於「讓任何人都能用一支影片取得賽事洞察」的目標，整合了電腦視覺分析 Pipeline、結構化數據展示、以及多模態大語言模型對話三大模組，打造一套開箱即用的網球賽事智慧分析平台。

---

### 二、作品功能簡介及特色

**核心功能：**

- **影片上傳**：支援分塊（Chunked）上傳，可處理大型影片檔案，並自動將 AV1/VP9 等格式轉碼為 H.264 以確保瀏覽器相容性
- **綜合分析**：單一流程整合球體偵測、球員姿態偵測、事件識別（擊球點、反彈、回合切換）、回合統計，輸出帶標注框的分析影片與 JSON 報告
- **五維分析視圖**：回合摘要、選手站位深度、球速統計、球場落點熱力圖、選手個別數據
- **AI 多模態聊天**：直接針對上傳影片提問，後端以串流方式即時回應，支援多輪對話並保留歷史記錄
- **歷史管理**：登入用戶可管理多部影片記錄，訪客模式七天自動清理

**產品特色：**

- 訪客免登入即可使用完整功能，大幅降低使用門檻
- 分析進度即時輪詢顯示（0–100%），使用者體驗流暢
- 深/淺色主題切換、Glass-morphism 現代化 UI 設計
- 一鍵 Docker Compose 部署，適合企業私有化佈署場景

### 三、開發工具與技術

| 層次 | 技術選型 |
|---|---|
| 前端 | Next.js 16（React 19、TypeScript）、Glass-morphism UI |
| 後端 | FastAPI（Python 3.x）、SQLAlchemy ORM、asyncio 非同步架構 |
| 資料庫 | MySQL 8.0 |
| AI 推理 | vLLM（OpenAI 相容 API）+ Qwen 多模態語言模型 |
| 影片處理 | FFmpeg、decord、OpenCV、Ultralytics YOLO（YOLOv11） |
| 部署 | Docker Compose + Cloudflare Tunnel（HTTPS） |
| 認證安全 | JWT（HS256）+ bcrypt，支援訪客 Token 機制 |

**技術獨特性：**

- **本地 LLM 推理**：以 vLLM 在自有 GPU 上運行多模態模型，不依賴第三方 API，保護使用者影片隱私，同時降低長期營運成本
- **思考鏈過濾（Chain-of-Thought Filtering）**：後端自動過濾模型 `<think>...</think>` 推理過程，僅將最終回應串流至前端，在保留模型推理品質的同時改善使用者體驗
- **並行 Chunk 上傳（Worker Pool）**：前端以 concurrency=3 的 XHR worker pool 進行平行分塊上傳，顯著提升大型影片檔案的上傳速度
- **In-memory Session 狀態管理**：分析進度透過 in-memory session dict 即時管理，避免頻繁 DB 寫入，提升進度回報的即時性

**與既有技術之差異化：**

相較於 Hawkeye、PlaySight 等商業系統，本平台具備以下差異化優勢：

| 比較項目 | 商業系統（Hawkeye 等） | 本平台 |
|---|---|---|
| 部署方式 | 專用硬體、廠商維護 | 自有 GPU + Docker，完全本地化 |
| 資料隱私 | 資料上傳至廠商雲端 | 資料不離開自有伺服器 |
| AI 互動 | 無自然語言對話功能 | 多模態 AI 對話，可針對影片提問 |
| 擴充彈性 | 封閉系統，難以客製 | 開放架構，可替換模型與新增分析模組 |
| 成本 | 高（授權費 + 設備費） | 低（僅硬體成本） |

**產品迭代與擴充規劃：**

系統架構採服務分層設計（routers → services → models），後續可無縫新增：
- 其他球類運動偵測模型（羽球、桌球、排球）
- 裁判線判輔助、戰術模式識別等進階分析模組
- 替換任意 OpenAI 相容的 AI 模型端點（持續跟進最新模型）
- 多語言介面支援

### 四、使用對象及環境

**主要使用對象：**

- 網球教練（賽事回顧、技術分析、訓練計劃制定）
- 職業／業餘選手（自我分析、賽後復盤、追蹤進步）
- 體育科學研究人員（運動數據收集與研究）
- 賽事主辦方（賽後自動化統計報告）

**軟體規格：**

| 項目 | 規格 |
|---|---|
| 伺服器作業系統 | Linux（Docker 環境） |
| 後端框架 | Python 3.10+、FastAPI、uvicorn |
| 前端框架 | Node.js 20、Next.js 16 |
| 資料庫 | MySQL 8.0 |
| AI 推理引擎 | vLLM nightly（需 CUDA GPU） |
| 影片格式支援 | MP4、MOV、AVI、MKV、WebM |
| 用戶端環境 | 任意現代瀏覽器（Chrome、Firefox、Edge、Safari），無需安裝任何軟體 |

**硬體規格（伺服器端）：**

| 項目 | 最低需求 | 建議規格 |
|---|---|---|
| GPU | NVIDIA GPU（CUDA 支援） | NVIDIA RTX 3090 或以上 |
| GPU 顯存 | 12 GB | 24 GB（用於 27B 以上參數模型） |
| RAM | 16 GB | 32 GB 以上 |
| 儲存空間 | 100 GB SSD | 500 GB SSD 以上 |
| 網路頻寬 | 100 Mbps | 500 Mbps 以上（支援大檔案上傳） |

### 五、產業應用性

**產業應用效益：**

- **訓練效率提升**：原本需要教練人工觀看 30 分鐘影片才能整理的數據，本平台可在數分鐘內自動輸出完整統計報告，顯著降低人力成本
- **科學化訓練依據**：球速統計、落點熱力圖、站位深度等量化指標，讓訓練計劃制定從「憑感覺」轉為「數據驅動」
- **AI 解說輔助決策**：多模態 AI 對話功能讓非技術背景的教練與選手也能從影片中獲得深度戰術洞察

**商業價值：**

- **SaaS 訂閱服務**：面向網球學院、俱樂部，按影片數量或分析時數計費的雲端訂閱模式
- **私有化佈署授權**：企業購買授權後部署於自有伺服器，保護比賽策略機密，適合職業球隊與國家隊
- **賽事數據服務**：提供賽後自動化統計報告給主辦方或體育媒體，降低人工數據整理成本

**社會影響力：**

- 降低高品質運動分析工具的取得門檻，使基層教練與學生選手也能享有過去僅限職業球隊的數據支援
- 推動台灣體育科技（SportsTech）發展，驗證 AI 技術在本土運動領域的應用可行性
- 促進運動訓練的數位轉型，提升台灣網球整體競技水準

**預計未來合作對象：**

- 各縣市網球協會與地方球隊
- 大學體育學院（運動科學研究合作）
- 網球學院與訓練中心
- 國內體育賽事主辦單位（如中華民國網球協會）
- 體育媒體與轉播平台

---

### 六、結語

本作品示範了如何將現代 AI 技術——電腦視覺物件偵測、多模態大語言模型、高效能非同步 API——整合為一套完整且可實際部署的運動分析平台。在架構上，系統強調模組化與可擴充性；在體驗上，強調低門檻（訪客免登入）與即時互動（串流回應、進度輪詢）；在安全性上，確保使用者影片資料全程不離開自有伺服器。

本平台不僅是技術整合的成果，更是對「AI 如何賦能運動訓練」這一命題的具體回答。未來將持續優化偵測精準度、擴展支援更多球類運動，並朝向 SaaS 商業化方向推進，期望成為台灣體育科技產業的標竿應用之一，協助更多教練與選手在訓練中獲得 AI 的力量。
