# 作品說明書
## 作品名稱：智慧網球比賽影片分析平台

## 作品摘要（500字內）
本作品為一套以人工智慧驅動的網球影片分析平台，使用者只需上傳一般比賽影片，系統即可自動完成網球、球場與球員偵測，進一步分析回合、擊球、球速、落點與站位，並以互動式圖表呈現。核心採用自訓練 YOLO 網球偵測模型與自訓練 YOLO 場地關鍵點偵測模型，搭配 Kalman 追蹤、Homography 世界座標轉換、VLM 輔助驗證，以及 RAG（檢索增強生成）問答技術，兼顧推論效率、分析準確性與解讀能力。平台採前後端分離與 Docker Compose 容器化部署，可應用於教練訓練、選手賽後檢討、校隊教學與運動科技服務，降低專業賽事分析的導入門檻，讓一般場館與基層球隊也能取得可量化、可追蹤的比賽洞察。

## 一、創意發想背景及概述
網球教練與選手在賽後檢討時，往往需要反覆觀看影片，人工紀錄每拍擊球、回合長度、站位變化與落點分布，流程耗時且難以標準化。現有高階運動分析工具多仰賴固定式攝影設備、感測器或高價商用系統，導致校隊、俱樂部與基層訓練單位不易導入。

本作品的發想，是希望只靠一段一般比賽影片，就能自動產出具有戰術價值的分析結果。平台整合電腦視覺、世界座標換算、VLM 輔助驗證與 RAG 問答流程，將原本分散且高度仰賴人工的賽後整理工作，轉化為可重複、可量化、可視覺化的數據服務，協助使用者更快掌握比賽內容與訓練方向。

## 二、作品功能簡介及特色
本平台的主要功能如下：

- 影片上傳後自動完成轉檔、分析與結果整理。
- 自動輸出標註影片與結構化分析資料，便於回看與追蹤。
- 提供五大分析面板，包含回合分析、球員統計、站位分析、速度統計與球場落點圖。
- 提供搭配 RAG（檢索增強生成）技術的 AI 對話介面，使用者可直接以自然語言詢問比賽內容與觀察重點。
- 支援歷史紀錄管理與訪客試用模式，降低首次使用門檻。

本作品的特色如下：

- 不需額外感測器或專用球場設備，只需一般錄影影片即可啟動分析。
- 可將球與球員位置投影到球場世界座標，讓球速、落點與站位具備實際量測意義。
- 以前端互動式視覺化呈現賽事資料，讓教練與選手能快速定位關鍵回合與問題球。
- 問答端搭配 RAG（檢索增強生成）技術，先檢索該場比賽的結構化分析資料，再交由模型生成回答，可降低幻覺並提升回答與比賽內容的對應性。

## 三、開發工具與技術
### 開發工具與架構
- 前端：Next.js 16、React 19、TypeScript
- 後端：FastAPI、SQLAlchemy、MySQL 8
- 影像處理：OpenCV、FFmpeg
- AI 與模型推論：Ultralytics YOLO、vLLM、Qwen 系列多模態模型、RAG（檢索增強生成）
- 部署：Docker Compose

### 技術獨特性
1. 自訓練 YOLO 球與場地偵測模型  
本系統使用自訓練 YOLO 網球偵測模型處理高速、小目標的網球追蹤問題，同時使用自訓練 YOLO 場地關鍵點偵測模型辨識球場 14 個關鍵點。相較直接套用通用模型，本作品更能適應網球場景中的小球體、拍攝角度變化與場地線干擾。

2. 世界座標換算與可量化分析  
系統利用場地關鍵點建立 Homography，將影像像素位置轉為真實球場公尺座標，進而計算球速、落點區域與球員站位深度。這使平台不只看得到影像結果，也能輸出具戰術與訓練價值的量化數據。

3. VLM 輔助驗證降低誤判  
平台先進行場地偵測、球員姿態辨識與球體追蹤，再以 Kalman 濾波、ROI 搜尋與靜止假陽性排除機制提升追蹤穩定度，最後使用 VLM 輔助驗證擊球事件與回合得分結果。這種「視覺偵測為主、VLM 驗證為輔」的設計，與完全依賴單一模型輸出的方式相比，更兼顧效率、可解釋性與準確率。

4. RAG 問答提升解讀能力  
系統會先將分析後的回合、球速、站位與落點等結構化資料整理為可檢索內容，再透過 RAG（檢索增強生成）技術提供問答服務，使模型回答能以該場比賽實際數據為基礎，而非僅依賴通用知識生成敘述。

### 與既有技術之差異化
- 相較需感測器、多鏡頭或固定場地設備的系統，本作品可由單一一般比賽影片啟動分析，導入成本較低。
- 相較只提供標註畫面的影像工具，本作品可進一步產出回合、速度、站位與落點等可視化統計。
- 相較純生成式 AI 解說，本作品先建立結構化賽事資料，再以 RAG（檢索增強生成）方式提供問答，可降低幻覺並提高回答與實際比賽的對應性。

### 產品／服務之迭代與擴充
- 第一階段：完成球體偵測、標註影片輸出與基本追蹤。
- 第二階段：加入場地關鍵點、世界座標換算、回合切割與球速分析。
- 第三階段：整合 VLM 輔助驗證、互動式分析面板與 RAG 問答。
- 後續規劃：擴充雙打場景、更多擊球型態辨識、即時串流分析、行動端服務與 API 對外授權。

## 四、使用對象及環境
### 使用對象
- 網球教練與培訓機構
- 校隊、俱樂部與業餘／競技選手
- 體育教學與運動科學研究單位
- 賽事單位、內容平台與運動科技業者

### 軟硬體規格
- 作業系統：Linux 伺服器環境，建議 Ubuntu 22.04 以上
- 容器與服務：Docker、Docker Compose、MySQL 8
- 前後端環境：Python 後端、Node.js 20 前端
- GPU：支援 CUDA 的 NVIDIA 顯示卡；若僅執行 YOLO 分析，建議 12GB VRAM 以上；若啟用 VLM 驗證或 RAG 問答模型，建議 24GB VRAM 以上或依模型大小調整
- CPU／記憶體：建議 8 核心以上 CPU、16GB RAM 以上，正式部署建議 32GB RAM
- 儲存空間：建議 100GB 以上 SSD，以利影片、模型與分析資料保存
- 使用端：支援現代瀏覽器，可透過網頁介面完成上傳、分析與查詢

## 五、產業應用性
### 產業應用效益
本作品可將原本高度仰賴人工的賽後影片整理流程，自動化為可複製的分析服務。對教練而言，可快速掌握球員在回合長度、擊球選擇、站位深度與失分模式上的問題；對選手而言，可更有效率地進行賽後修正；對學校與俱樂部而言，則可在有限預算下導入數據化訓練流程。

### 商業價值
- 可發展為教練、俱樂部與學校採用的 SaaS 訂閱服務。
- 可提供企業版或地端部署方案，供協會、培訓中心與賽事單位使用。
- 可將分析結果 API 化，提供運動科技平台、內容平台或賽事系統整合。
- 可延伸為選手追蹤資料庫與訓練成效管理服務，提升長期客戶黏著度。

本作品有助於降低專業運動分析工具的使用門檻，讓基層球隊與校園訓練也能使用 AI 進行客觀分析，促進運動科技普及、提升訓練效率，並帶動國內體育數位化與資料化發展。

本作品以「讓一般影片也能轉化為可用的網球數據」為核心目標，結合自訓練 YOLO 球與場地偵測模型、世界座標換算、VLM 輔助驗證、RAG 問答與互動式前端分析，打造一套兼具技術深度與產業落地性的智慧運動分析平台。未來將持續優化模型表現、擴充服務型態與合作場域，朝更即時、更普及、更具商業規模的運動 AI 服務發展。
